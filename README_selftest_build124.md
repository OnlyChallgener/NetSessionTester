# NetSessionTester V1.1.15 build124 自测说明

本包不作为正式发版，仅用于验证 Ping 图表交互与高频显示策略。

## 改动

- Ping 图形统一为延迟波形/折线图。
- 间隔 < 200ms 自动按高频处理；间隔 >= 200ms 自动按低频处理，不再额外增加高频开关。
- 高频模式后台保留更细采样，图表按 250ms 桶聚合显示，降低 UI 抖动。
- 正常延迟用蓝色折线连接。
- 丢包不参与延迟连线，使用红点/红线/淡红色区块表示。
- 连续或密集丢包自动合并为淡红色区块，避免手机小屏糊成一片。
- 图表支持左右拖动查看历史，双击或点“回实时”恢复实时跟随。
- 点击曲线显示当时或聚合区间的数据。
- X/Y 轴标签动态控制数量，避免拥挤和遮挡。

## 重点自测

1. 25ms / 50ms / 100ms 是否自动显示“自动高频”。
2. 200ms / 500ms / 1000ms 是否显示“自动低频”。
3. 丢包是否显示为红色标记/区块，而不是被当成延迟线连接。
4. 左右拖动图表后是否进入历史查看。
5. 双击图表或点“回实时”是否回到最新。
6. 点击图表是否显示当时数据。
7. X 轴和 Y 轴文字是否不拥挤、不遮挡。
