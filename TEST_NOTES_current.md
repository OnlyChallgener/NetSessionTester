# NetSessionTester v0.9.9 test89 自测版

用途：仅用于自测 9000-10000 会话区间卡住问题，不建议作为正式发布包直接发布。

## 改动

- versionCode = 89
- versionName = v0.9.9-test89
- update.json 暂时保持 build88，避免未发布 v89 release 时 App 检测更新出现 404。
- TCP 发射核心继续使用 100ms tick 流水线。
- 扩大 pending connect 窗口：maxInFlight = targetCps * 8，范围 2000 - 16000。
- pending 仍计入 FD 预算，达到 32360 左右会裁剪并释放。
- 失败阈值改为四档：
  - 0 - 1000：120
  - 1000 - 6000：200
  - 6000 - 12000：360
  - 12000+：600
- 6000 - 12000 区间内，360 作为软阈值；达到后做 2 秒增长确认。
  - 2 秒成功增长 < 60：释放
  - 仍有明显增长：继续发射
  - 失败达到 600：硬释放
- 取消失败曲线、CPS曲线、3s/5s无增长检测保持不变。

## 重点观察

1. 使用 baidu:80 / taobao:443 复测是否仍停在 9000-10000。
2. 如果可以突破 12000，说明 build88 的 360 硬切断是主要原因之一。
3. 如果仍停在 10000 左右，重点看失败区间和失败原因：超时 / 重置 / 端口耗尽 / FD上限。
4. 如果 pending 过多造成卡顿，可把 maxInFlight 从 targetCps * 8 降到 *6。
