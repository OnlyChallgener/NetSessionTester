# NetSessionTester v0.9.9 build 71 本地自测说明

本版本用于自测，不修改 update.json，不作为推送版本。

## 重点改动

- 新增低容量快速确认：低会话数网络在失败率升高且增长缓慢时进入短确认，避免长时间低 CPS 占用。
- 新增平台期增长效率判断：连接数到达一定值后，如果最近窗口增长效率偏低，会进入顶部确认。
- 顶部确认 CPS 保持约 16 - 32，并增加低 CPS 硬时间上限，避免 16 - 32 CPS 长时间慢跑。
- 用户修改“新增值”后，该值作为手动 CPS 上限，测试仍然平滑起步，失败、无增长、FD 保护仍会自动降速或停止。
- 优化 CPS 曲线绘制：CPS 采样点存在较大时间缺口时不再用长斜线连接，减少误导。

## 建议测试点

1. 低容量网络：观察是否在十几秒内进入确认并结束，失败数不再大幅堆积。
2. 10000 左右平台：观察到达平台后是否不再长时间 16 - 32 CPS 慢跑。
3. 手动新增值 1000：观察是否作为上限生效，而不是一开始就硬冲 1000 CPS。
4. 会话数 & CPS 双轴图：观察 CPS 线是否不再出现长距离误导性斜线。
