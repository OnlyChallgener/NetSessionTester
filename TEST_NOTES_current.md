
# V1.1.14 build117 自测重点
1. 首页小卡片、工具页按钮、下拉按钮点击时水波纹应为圆角，不应出现方形扩散。
2. 长按拖拽卡片时阴影应尽量跟随圆角扩散。
3. MTU 检测分别测试“互补 / 快速估测 / PMTU增强”，观察检测过程是否能显示 PMTU/DF 或快速回退信息。
4. 手机断开 WiFi 使用蜂窝网络时，漫游测试点击开始应提示“请连接WIFI”，且不启动测试。
5. 漫游测试开始后应显示 Ping表 和 信号表，X/Y 轴有数字刻度。
6. 点击曲线应显示当时采样点的网关/外网延迟、RSSI、速率信息。
7. 走动触发 BSSID 切换后，应自动出现漫游详情小卡片。
8. 停止测试后，应显示最大/最小/平均延迟、最大/最小/平均信号、协商速率和漫游切换总结。

# NetSessionTester V1.1.14 build116 自测说明

此包为自测包，不发版。重点验证：

1. 首页网络信息区域固定展示 NSLookup / Tracket / MTU检测 / 漫游测试 四个小卡片。
2. 点击 MTU检测进入二级页面，系统返回和左上角返回都回 APP。
3. MTU检测可显示二分探测过程、最终 MTU 估测和分析。
4. IPv6 MTU 检测时，域名解析应优先使用 AAAA；无 AAAA 时显示解析失败。
5. 点击漫游测试进入二级页面，返回仍在 APP。
6. 漫游测试可显示 RSSI、协商速率、BSSID、网关/外网延迟、丢包和曲线。
7. NSLookup 页面可切换本机DNS / 自定义DNS。
8. 自定义DNS模式下 DNS1 无记录、超时或失败时继续尝试 DNS2。
9. 点击 NAT类型 小卡片进入 NAT 诊断。
10. 长按拖动卡片、点击卡片时阴影/水波纹应为圆角，不应出现明显方形阴影。

限制说明：
- MTU 是基于系统 ping 能力的路径 MTU 估测，不是底层绝对 PMTU。
- 漫游测试的 BSSID/RSSI 读取需要系统 WiFi/定位权限，部分系统会隐藏或限制。
- WiFi 扫描频率受 Android 系统限制，本版只做当前连接实时采集，不做 AP 列表扫描。
