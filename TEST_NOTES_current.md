# v0.9.9 build84 performance-fix2

本包针对实测反馈继续修复：

- 网络信息卡片右下角恢复显示“运营商”，底部标签也恢复“运营商 xxx”。
- 旧设置里保存过 128 CPS 时，升级后自动迁移为 1000 CPS，避免继续显示 128/s。
- 会话核心改为高速批量窗口，不再按 100ms 小批次等待整批 connect 完成，目标是恢复接近图4的 30~40s 级别 3万会话性能。
- 继续取消 CPS 曲线，只绘制蓝色会话曲线。
- 保留 Ping 图表刷新。
- 图表下方增加失败区间摘要，例如：失败区间：0-10s +263。
- 保留 FD 安全保护，接近 Android FD/Socket 上限时停止新增并释放，避免 32500+ 闪退。

建议 GitHub Actions 编译后测试：

```bash
./gradlew clean
./gradlew assembleRelease
```


## build84 performance-final 测试重点

1. 默认目标 CPS 应显示 200/s；手动改 500/800/1000 后应保留。
2. 右下网络卡片应显示 NAT置信度；底部状态标签应显示运营商。
3. 0 失败时应接近 32360 后立即释放，不再额外等待。
4. 低会话失败应在约 120 次附近收尾；1000-6000 为 200，6000-12000 为 300，12000 以上为 600。
5. 中高会话阶段不做无增长终止，只看失败区间和 FD 保护。
