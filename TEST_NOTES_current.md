# v0.9.9 build80 自测说明

本地自测版，不更新 update.json。

## 核心调整

1. 失败数改为按峰值分段上限：
   - peak < 1000：失败上限 120
   - peak < 6000：失败上限 200
   - peak < 12000：失败上限 360
   - peak >= 12000：失败上限 600

2. 低容量快停：
   - 运行 >= 6 秒
   - 峰值 < 1000
   - 最近 4 秒增长 < 30
   - 失败 >= 80
   - 直接停止新增并释放，避免几百会话僵直十几秒。

3. TCP 超时固定 3000ms，继续不使用动态超时。

4. FD 预警不再降速：
   - 去掉 FD 预警区/保护区的降 CPS 逻辑
   - 接近 FD 上限直接停止新增并释放
   - 减少高容量阶段“降速一下就卡顿”的问题。

5. 手动 CPS 仍优先按用户设定发射。
