# v0.9.9 build62 本地测试版：后台继续增长修复

本地测试用，不用于发布。

## 修复内容

1. 测试核心循环不再绑定 Activity / Compose 生命周期
   - 新增 `AppTestRuntime` 应用级协程作用域。
   - TCP 会话增长循环改为运行在 `Dispatchers.Default` 的应用级作用域。
   - UI 只负责接收状态更新和展示，不再驱动真正的新增连接循环。

2. 增强前台服务后台保活
   - 前台服务启动时申请 `PARTIAL_WAKE_LOCK`。
   - 测试结束、强制释放、服务销毁时释放 WakeLock。
   - Manifest 增加 `WAKE_LOCK` 权限。

3. 避免后台时 UI 更新阻塞测试循环
   - 统计和日志通过 `AppTestRuntime.mainScope` 异步投递到主线程。
   - 测试循环不等待 Compose 重绘，避免后台时新增连接停住。

## 验证方法

1. 开始 IPv6 或分别测试。
2. 等会话数增长到几千或一万以上。
3. 按 Home 键切后台 60 秒。
4. 回到 App 查看增长曲线。

预期：后台期间连接数继续增长，不再出现长时间水平平台。

## 注意

小米 / 澎湃 OS / HyperOS 仍可能需要手动设置：
- 电池策略：无限制
- 允许后台运行
- 允许前台服务通知
- 多任务界面锁定 App
