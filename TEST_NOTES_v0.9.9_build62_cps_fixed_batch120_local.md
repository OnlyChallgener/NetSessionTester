# NetSessionTester v0.9.9 build62 本地测试版：CPS 固定节奏与默认新增 120

本版本仅用于个人测试，不用于正式发布。

## 改动

1. CPS 调速策略回退为固定节奏
- 删除失败率/连续无成功后的 10 CPS、1 CPS、2 CPS 长时间低速巡航逻辑。
- 增长测试按用户设置的 batch / interval 持续新增连接。
- 危险场景不再降速拖延，而是直接硬终止并进入统一释放。

2. 失败上限硬终止
- 失败数达到用户设置的失败停阈值后，立即结束当前协议测试。
- 例如失败停=200，失败数达到 200 后显示“失败上限”，并统一释放、保存历史。
- 失败数接近上限时不再把新增数限制到剩余失败数，避免失败 199 时 CPS 卡在 1/2。

3. FD / 资源保护逻辑保持
- FD 保护、资源保护、网络切换、用户停止仍然走统一释放。
- 不修改 releaseAndFinalize / generation 防回流逻辑。

4. 默认起始新增改为 120
- 新安装默认 batchSize 从 100 改为 120。
- 恢复默认参数时也会恢复为 120。
- 已有安装如果本地保存过设置，仍可能读取旧值，需要手动改为 120 或恢复默认。

5. STUN 服务器保持上一版 8 个节点
- 未改动 NAT / DNS / UI 其他逻辑。
